    Scorecard Legend - Adapted from "7 Cs for Synthetic Medical Data Evaluation" (Zamzmi et al., 2025)
    Column Scorecard Metric Paper Metrics Used Description
    Entity - - The medical entity text being evaluated
    Type - - Entity classification: disease, symptom, finding, organ, imaging_procedure, examination_procedure, therapeutic_procedure, imaging_result, examination_measure, parameter, score, therapy, substance, adverse_event
    Cong. 1. Congruence Cosine Similarity, Distance to Centroid (Embedding space, Maximize) Semantic alignment with KG. Uses embedding similarity to find nearest entity. High = matches terminology.
    Cov. 2. Coverage Recall, Coverage, Clustering-Based Metrics (Embedding space, Maximize) Gap-filling assessment. Counts similar entities, checks if novel. High = valuable addition.
    Const. 3. Constraint Constraint Violation Rate, Distance to Constraint Boundary (Minimize violations) Clinical validity. Checks format, length, characters, medical plausibility. High = valid concept.
    Column Scorecard Metric Paper Metrics Used Description
    Comp. 4. Completeness Proportion of Required Fields, Missing Data % (Metadata, Maximize/Minimize) Required attributes present: type, definition, normalized form, context, confidence.
    Cons. 5. Consistency Variance, Max-Min Difference (Quality Metrics, Minimize variance) Cross-validation agreement. Compares LLM vs BERT classification. High = classifiers agree.
    Overall Weighted Score Congruence (25%) + Constraint (25%) + Consistency (20%) + Coverage (15%) + Completeness (15%)
    Status Review Status ✓ Passed (≥75%) | ⚠ Review (50-74%) | ✗ Failed (<50%)
    Weight Justification
    • Congruence (25%) - High weight: Semantic alignment with existing KG terminology is crucial for interoperability and consistency.
    • Constraint (25%) - High weight: Clinical validity is paramount in medical data; invalid entities can cause patient safety issues.
    • Consistency (20%) - Medium-high: Agreement between independent classifiers (LLM + BERT) indicates reliable classification.
    • Coverage (15%) - Lower weight: Novelty is valuable but not critical; redundant valid entities are acceptable.
    • Completeness (15%) - Lower weight: Missing optional fields (context, definition) don't invalidate the core entity.
