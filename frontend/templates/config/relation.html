Parameter Documentation
Parameter Description Formula / Usage Example
Congruence (Semantic Validity)
Type Penalty Score penalty when the relation type is not semantically valid between the given entity types. If (src_type, tgt_type) not in valid combos → score -= penalty substance → has_symptom → disease is invalid → penalty 0.3 applied
KG Bonus Extra bonus when a similar relation already exists in the Knowledge Graph. $score = base + bonus \times 0.5$ if KG match Base 0.9 + Bonus 0.1 = 1.0 for KG-confirmed
Coverage (Representativeness)
Novelty Threshold Maximum number of similar relations in KG to consider the new one as "novel" (fills a gap). If similar_count < threshold → relation is novel Threshold=3: If 1 similar → ✓ Novel; If 5 → ✗ Redundant
Novelty Bonus Bonus added to Coverage score when the relation fills a gap in the KG. $score_{coverage} = 0.5 + novelty\_bonus$ if novel Base 0.5 + Bonus 0.3 = 0.8 for novel relation
Constraint (Domain Rules)
Violation Penalty Score penalty for each domain-rule violation (invalid types, self-reference, reversed directionality, etc.). $score = 1.0 - (n_{violations} \times penalty)$ 2 violations × 0.25 = -0.5 → Final: 0.5
Min Confidence Minimum classification confidence to accept without penalty. If confidence < min → violation triggered Min=0.3: Confidence 0.2 → ⚠ penalty; 0.8 → ✓ OK
Require Context Whether a text context is mandatory for the relation. If true and context is empty → half-penalty Required + missing → -0.125 penalty
Completeness (Required Attributes)
Required Fields List of mandatory fields for a relation to be considered complete. Possible values: source_entity, target_entity, relation_type, source_type, target_type, confidence, context. $score_{required} = \frac{fields\_present}{total\_required}$
Weight: (1 - optional_weight) of total 		Required: 5 fields, present: 5 → 1.0
Optional Weight Relative weight of optional fields (confidence, context) in the Completeness score. $score = (score_{req} \times 0.85) + (score_{opt} \times 0.15)$ Req=1.0, Opt=0.5 → (1.0×0.85) + (0.5×0.15) = 0.925
Consistency (Cross-Validation)
Agreement Bonus Bonus when both LLM and BERT classifiers detect the same relation. Indicates higher confidence. $score = confidence + bonus$ if both agree
$score = \min(0.85, confidence)$ if single source 		Both: confidence 0.7 + bonus 0.3 = 1.0
Base Score Default consistency score when confidence data is not available. $score = confidence \text{ or } base\_score$ Without confidence → uses Base Score 0.5
Overall Score Formula

The final relation score is calculated as the weighted sum of the 5 metrics, adapted from the "7 Cs for Synthetic Medical Data Evaluation" (Zamzmi et al., 2025):

$Score_{total} = \sum_{i=1}^{5} (score_i \times weight_i)$

Expanded:

Score = (Congruence × W₁) + (Coverage × W₂) + (Constraint × W₃) + (Completeness × W₄) + (Consistency × W₅)

Example with default weights:

A relation with scores: Congruence=0.90, Coverage=0.80, Constraint=1.00, Completeness=0.93, Consistency=0.70
Score = (0.90×0.25) + (0.80×0.15) + (1.00×0.25) + (0.93×0.15) + (0.70×0.20)
Score = 0.225 + 0.12 + 0.25 + 0.1395 + 0.14 = 0.87 (87%) → ✓ Passed
