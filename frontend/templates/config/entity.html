Parameter Documentation
Parameter Description Formula / Usage Example
Congruence (Semantic Alignment)
Min Similarity Minimum similarity threshold to consider an entity as a valid match in the Knowledge Graph. If similarity ≥ min_similarity → valid match With 0.7: "diabetes mellitus" with 0.85 similarity to "diabetes" → ✓ Match
Exact Match Bonus Extra bonus when the entity exists exactly in the KG. $score = base\_score + bonus$ if exact match Base 0.8 + Bonus 0.2 = 1.0 for exact match
Coverage (Representativeness)
Novelty Threshold Maximum number of similar entities to consider the new one as "novel" (fills a gap in the KG). If similar_count < threshold → entity is novel Threshold=3: If 2 similar entities → ✓ Novel; If 5 → ✗ Not novel
Novelty Bonus Bonus added to Coverage score when the entity is novel (fills a gap in the KG). $score_{coverage} = 0.5 + novelty\_bonus$ if novel Base 0.5 + Bonus 0.3 = 0.8 for novel entity
Constraint (Clinical Validity)
Min/Max Length Valid character range for entity text length. min ≤ len(entity_text) ≤ max Min=2, Max=200: "T2DM" (4 chars) → ✓ Valid
Violation Penalty Score penalty for each constraint violation detected. $score = 1.0 - (n_{violations} \times penalty)$ 2 violations × 0.25 = -0.5 → Final score: 0.5
Completeness (Required Attributes)
Required Fields List of mandatory fields the entity must have to be considered complete. Possible values: type, text, context, definition, normalized_form, confidence. $score_{required} = \frac{fields\_present}{total\_required}$
Weight: 70% of total Completeness score 		Required="type,text,context"
Entity has: type ✓, text ✓, context ✗
Required score: 2/3 = 0.67
Optional Weight Relative weight of optional fields (not listed in Required) in the Completeness score calculation. $score_{total} = (score_{req} \times 0.7) + (score_{opt} \times opt\_weight)$
Where $opt\_weight$ is the configured value (default 0.30) 		Score_req=0.67, Score_opt=1.0, Opt_weight=0.30
Total: (0.67×0.7) + (1.0×0.3) = 0.77
Consistency (Type Coherence)
Agreement Bonus Bonus when the LLM classifier and BERT classifier agree on the entity type. Indicates higher classification confidence. $score = confidence + agreement\_bonus$ if LLM = BERT
$score = confidence - (bonus \times 0.75)$ if LLM ≠ BERT 		LLM: "disease", BERT: "disease" → Agreement ✓
Confidence 0.7 + Bonus 0.3 = 1.0
Base Score Default consistency score when confidence is not available or cross-validation was not executed. $score_{initial} = confidence \text{ or } base\_score$ Without confidence → uses Base Score 0.5
Overall Score Formula

The final entity score is calculated as the weighted sum of the 5 metrics:

$Score_{total} = \sum_{i=1}^{5} (score_i \times weight_i)$

Expanded:

Score = (Congruence × W₁) + (Coverage × W₂) + (Constraint × W₃) + (Completeness × W₄) + (Consistency × W₅)

Example with default weights:

An entity with scores: Congruence=0.85, Coverage=0.80, Constraint=0.90, Completeness=0.75, Consistency=0.95
Score = (0.85×0.25) + (0.80×0.15) + (0.90×0.25) + (0.75×0.15) + (0.95×0.20)
Score = 0.2125 + 0.12 + 0.225 + 0.1125 + 0.19 = 0.86 (86%) → ✓ Passed
